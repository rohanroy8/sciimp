# Phase 04: Compression Pipeline - Plan

## Goal
Integrate RLE, zstd compression, and delta frames with periodic keyframes into both the Encoder and Player to drastically reduce `.ascv` file sizes and maintain fast rendering.

## Artifacts this phase produces
- `include/ascv/format.hpp` updated with `FrameType` and `FrameHeader`.
- CMake configurations updated to link `zstd`.
- Encoder and Player updated with compression logic.

## Verification
- Running the Encoder CLI produces `.ascv` files that are significantly smaller than the raw uncompressed format from Phase 3.
- The Player successfully plays back these compressed `.ascv` files with the correct visual output and frame rate.

<task>
  <read_first>
    - src/encoder/EncoderCore.hpp
    - src/encoder/EncoderCore.cpp
  </read_first>
  <action>
    Update the Encoder to implement delta encoding, RLE, and zstd compression.
    1. Add an `int frame_index_` counter and a `std::vector<char> previous_frame_` buffer to the `EncoderCore` class.
    2. Introduce a custom, fast RLE encoding function (e.g., `std::vector<uint8_t> compress_rle(const std::vector<char>& data)`). A simple `[count_8bit][byte_value_8bit]` format is sufficient.
    3. Modify the frame processing loop:
       - Every 30 frames (e.g., `frame_index_ % 30 == 0`), treat as an I-frame.
       - For P-frames, compute a delta: compare current frame to `previous_frame_`. If a character matches, output `\0`; else output the new character.
       - Apply RLE to the I-frame buffer or P-frame delta buffer.
       - Apply Zstd compression to the RLE output using `ZSTD_compress()`.
    4. Write a `FrameHeader` (with correct type and `compressed_size`), followed by the Zstd-compressed payload to the output `.ascv` file.
    5. Update `previous_frame_` with the current raw frame.
  </action>
  <acceptance_criteria>
    The encoder compiles and successfully writes heavily compressed `.ascv` files without crashing.
  </acceptance_criteria>
</task>

<task>
  <read_first>
    - src/player/main.cpp
  </read_first>
  <action>
    Update the Player to read `FrameHeader`s, decompress zstd and RLE data, and apply delta frames.
    1. Read the `FrameHeader` from the input stream.
    2. Read `compressed_size` bytes into a buffer.
    3. Use `ZSTD_decompress()` to retrieve the RLE-compressed data.
    4. Apply custom RLE decompression to retrieve the raw frame buffer (for I-frame) or delta buffer (for P-frame).
    5. Maintain a persistent `std::vector<char> current_frame_` buffer across iterations.
    6. For an I-frame, copy the decompressed data directly into `current_frame_`.
    7. For a P-frame, apply the delta buffer to `current_frame_` (only replace characters where the delta byte is not `\0`).
    8. Write `current_frame_` to the terminal.
  </action>
  <acceptance_criteria>
    The player successfully plays back the new compressed `.ascv` files at the correct frame rate, matching the original visual output.
  </acceptance_criteria>
</task>
