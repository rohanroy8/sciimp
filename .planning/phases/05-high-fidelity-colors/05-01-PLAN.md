# Phase 05: High Fidelity Colors - Plan

## Goal
Upgrade the rendering pipeline to support 16/256 ANSI colors and 24-bit True Color RGB output.

## Artifacts this phase produces
- Updated `include/ascv/format.hpp` with `ColorMode`.
- Updated `src/encoder/main.cpp` with color CLI choices.
- Updated `EncoderCore` with RGB frame extraction and color-matching math.
- Updated `ascv_player` to emit ANSI escape codes cleanly with color state tracking.

## Verification
- Running `ascv_encoder` with `--color-mode truecolor` generates a valid `.ascv` file.
- Running `ascv_player` on it renders colored output in the terminal.
- Test suites build and run successfully.

    5. Update frame buffer allocations to `W * H * S`.
    6. Implement delta comparison block-by-block. For each cell of size `S`, if all bytes match `previous_frame`, set the cell in `delta_frame` to all `\0`. Otherwise, copy the cell.

  <action>
    Modify `src/player/main.cpp` to decompress and render colored ANSI output efficiently.
    1. Read `color_mode` from the updated file header.
    2. Determine cell size `S` based on `color_mode`.
    3. Decompress the stream using `S` bytes per cell.
    4. Maintain `current_frame_` of size `W * H * S` and apply delta bytes correctly cell-by-cell.
    5. In the rendering loop:
       - Keep track of `last_fg_color` to prevent duplicate ANSI sequences.
       - If monochrome, write char.
       - If ANSI 16/256, write color change sequence `\x1b[38;5;<index>m` only if index changes.
       - If True Color, write color change sequence `\x1b[38;2;<R>;<G>;<B>m` only if color changes.
       - Reset terminal color at the end of the frame/player exit using `\x1b[0m`.
  </action>
