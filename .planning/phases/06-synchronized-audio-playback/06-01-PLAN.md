# Phase 06: Synchronized Audio Playback - Plan

## Goal
Implement synchronized audio playback using MiniAudio, mapping the player's video frame pacing dynamically to the active audio device clock.

## Artifacts this phase produces
- Updated `CMakeLists.txt` linking `libswresample`.
- Updated `EncoderCore` to extract, decode, resample, and write audio tracks to a sidecar WAV file (`.ascv.wav`).
- Updated `ascv_player` linked against `miniaudio` to play the sidecar WAV file and sync frames to the audio timeline.

## Verification
- Running the encoder against a video with audio produces both the `.ascv` file and a matching `.ascv.wav` sidecar file.
- Playing the output using `ascv_player` outputs sound synchronized with the text animation.
- If audio is missing, the player gracefully falls back to system timer timing.

  <action>
    Update `EncoderCore` to extract and resample the audio stream from the input video and write it to a sidecar `.wav` file.
    1. In `FfmpegContext`, search for the audio stream index (`AVMEDIA_TYPE_AUDIO`).
    2. If an audio stream exists:
       - Set up the AVCodecContext for audio decoding.
       - Initialize a `SwrContext` (`swr_alloc_set_opts2` or `swr_alloc` + `opt_set` configurations) to resample the audio to 16-bit signed PCM, stereo (2 channels), 44100Hz sample rate.
    3. During the frame decoding loop in `encode()`:
       - Decode audio packets.
       - Resample decoded frames using `swr_convert()`.
       - Write resampled PCM bytes to a temporary output buffer.
    4. At the end of encoding, write a standard 44-byte WAV header followed by the accumulated PCM bytes to `output_path + ".wav"`.
  </action>
  <acceptance_criteria>
    Encoding a video containing an audio stream creates a valid, playable `.ascv.wav` file alongside the `.ascv` file.
  </acceptance_criteria>
</task>

<task>
  <read_first>
    - src/player/main.cpp
  </read_first>
  <action>
    Modify `src/player/main.cpp` to play the sidecar WAV file using MiniAudio and synchronize the rendering loop.
    1. Implement a MiniAudio playback session in `main.cpp`.
    2. Determine the path of the sidecar WAV file: `wav_path = ascv_path + ".wav"`. Check if it exists.
    3. If the WAV file exists:
       - Initialize the `ma_engine` and load the WAV file as a `ma_sound` instance.
       - Start audio playback.
       - In the main video rendering loop:
         - Query the current audio position in seconds using `ma_sound_get_cursor_in_seconds()`.
         - Determine the target frame: `target_frame = static_cast<uint32_t>(cursor_seconds * (fps_numerator / fps_denominator))`.
         - Loop from the current frame index up to `target_frame` to read and apply P-frame delta updates sequentially.
         - Draw the frame.
         - Pace the rendering loop by calculating how long to sleep/spin until the next frame's audio timestamp.
    4. If the WAV file does not exist, fall back to the existing steady_clock pacing code.
    5. Clean up MiniAudio resources (`ma_sound_uninit`, `ma_engine_uninit`) on player exit.
  </action>
  <acceptance_criteria>
    Playing an `.ascv` file with a sidecar `.wav` renders the video in sync with the audio track. Pressing Ctrl+C halts audio playback instantly and exits cleanly.
  </acceptance_criteria>
