# Phase 07: Seeking/Scrubbing - Plan

## Goal
Implement interactive playback controls (seeking forward/backward, pausing/resuming) in the player, keeping the audio and video streams synchronized.

## Artifacts this phase produces
- Updated `src/player/terminal.cpp` to enable non-blocking raw mode on standard input.
- Updated `src/player/main.cpp` implementing the interactive loop with frame offset indexing, pause states, and audio sound cursors seeking.

## Verification
- Pressing `Space` pauses and resumes both video and audio.
- Pressing `Right Arrow` or `d` / `l` seeks forward 5 seconds; the video jumps and sound matches instantly.
- Pressing `Left Arrow` or `a` / `h` seeks backward 5 seconds; the player correctly navigates to the closest keyframe, rebuilds the state, and sound syncs.
- Pressing `q` terminates playback gracefully.

<task>
  <read_first>
    - src/player/main.cpp
  </read_first>
  <action>
    Modify `src/player/main.cpp` to index all frame offsets on startup and support interactive seeking and pausing.
    1. Implement a non-blocking key read helper `bool get_key_nonblocking(char& ch)` using standard C++ or POSIX `read(STDIN_FILENO, &ch, 1)`.
    2. Implement an in-memory frame index scan:
       - Save the starting stream file position.
       - Read through all frames: record `offset`, `type`, and `compressed_size` in a `std::vector<FrameInfo>` index.
       - Seek forward by `compressed_size` for each frame.
       - Restore the file stream to the first frame.
    3. Modify the playback loop to check for key events:
       - `q` sets `ascv::g_shutdown_requested = 1`.
       - `Space` toggles a `paused` state. When paused, stop audio via `ma_sound_stop(&sound)`. On resume, start it via `ma_sound_start(&sound)`.
       - `Right Arrow` (handled by parsing ANSI escapes `\x1b[C` or checking `d` / `l`) and `Left Arrow` (`\x1b[D` or `a` / `h`) calculate a new target frame index (`current_decoded_frame_idx +/- (5 * fps)`). Clamp between `0` and `frame_count - 1`.
    4. Implement seeking execution:
       - Find the closest preceding `I_FRAME`.
       - Seek stream to the keyframe offset.
       - Decode the keyframe, then decode all intervening P-frames up to the target frame index (skip drawing to terminal during sync).
       - Render the target frame.
       - Seek audio cursor: `double t = target_frame * (static_cast<double>(header.fps_denominator) / header.fps_numerator);` followed by `ma_sound_seek_to_pcm_frame(&sound, static_cast<ma_uint64>(t * 44100))`.
    5. Clean up MiniAudio resources on premature exit.
  </action>
  <acceptance_criteria>
    The player successfully plays back video, responds instantly to pause/resume commands, and correctly seeks forward/backward in synchronization with the audio.
  </acceptance_criteria>
</task>
